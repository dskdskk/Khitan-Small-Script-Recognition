Dataset Generation Scripts

This folder provides scripts for constructing Khitan Small Script synthetic datasets from annotated rubbing images and sign crop banks. All machine-specific paths have been removed. Users should provide input and output paths through command-line arguments.

The dataset generation pipeline contains four main steps:

1. Build a real sign crop bank from annotated rubbing images.
2. Generate random synthetic layouts from the crop bank.
3. Generate sequence-based synthetic samples using real Khitan character sequences.
4. Generate degradation-level datasets for ablation experiments.

All output annotations are saved in normalized center-size bounding box format:

```text
[c_x, c_y, w, h]
```

where `(c_x, c_y)` denotes the normalized box center, and `(w, h)` denotes the normalized box width and height.

------

## 1. Build Radical Bank

This script crops individual signs from annotated rubbing images and organizes them into class-wise folders.

```bash
python build_radical_bank.py \
  --input-dir dataset\test_239 \
  --output-dir data/radical_bank \
  --center-format \
  --clear-output
```

### Input

`--input-dir` should contain annotated rubbing images and their corresponding JSON annotation files.

The annotation file can be named in either of the following forms:

```text
image_name.png.json
image_name.json
```

Each annotation item should contain a class ID and a bounding box.

### Output

The cropped signs are saved by class ID:

```text
data/radical_bank/
├── 0/
│   ├── sample_1.png
│   └── ...
├── 1/
│   ├── sample_1.png
│   └── ...
└── ...
```

### Main arguments

| Argument             | Description                                            |
| -------------------- | ------------------------------------------------------ |
| `--input-dir`        | Directory containing annotated images and JSON files   |
| `--output-dir`       | Output directory for the cropped sign bank             |
| `--center-format`    | Treat input boxes as `[c_x, c_y, w, h]`                |
| `--no-center-format` | Treat input boxes as `[x, y, w, h]`                    |
| `--clear-output`     | Remove the existing output directory before processing |

## 2. Random Dynamic Layout Synthesis

This script randomly samples signs from the crop bank and places them into unified one- to seven-sign layout templates. It is used to generate synthetic samples with diverse sign combinations and rubbing-style appearances.

```bash
python synthesize_dynamic_layout.py \
  --bank-dir dataset/radical_bank \
  --output-root dataset/synth_dynamic \
  --num-images 130000 \
  --clear-output
```

### Input

`--bank-dir` should point to the crop bank generated by `build_radical_bank.py`.

### Output

The generated dataset is saved in train/val/test format:

```text
data/synth_dynamic/
├── train/
│   ├── images/
│   └── labels.json
├── val/
│   ├── images/
│   └── labels.json
└── test/
    ├── images/
    └── labels.json
```

### Main arguments

| Argument         | Description                               |
| ---------------- | ----------------------------------------- |
| `--bank-dir`     | Input sign crop bank                      |
| `--output-root`  | Output dataset directory                  |
| `--num-images`   | Number of synthetic images to generate    |
| `--img-size`     | Output image size, default is `256`       |
| `--block-min`    | Minimum character block size              |
| `--block-max`    | Maximum character block size              |
| `--train-ratio`  | Training split ratio, default is `0.95`   |
| `--val-ratio`    | Validation split ratio, default is `0.04` |
| `--seed`         | Random seed for reproducibility           |
| `--clear-output` | Remove existing output before generation  |

## 3. Real-Sequence Layout Synthesis

This script generates synthetic samples according to real Khitan Small Script character sequences. Unlike random synthesis, the sign IDs are determined by the input sequence file.

```bash
python create_real_layout_dataset.py \
  --bank-dir dataset/radical_bank \
  --output-root dataset/synth_real_layout \
  --radicals-file assets/radicals.txt \
  --sequences-file dataset/sequences.csv \
  --augmentations-per-seq 10 \
  --clear-output
```

### Input

`--radicals-file` defines the mapping between characters and class IDs. The character index in this file is used as the class ID.

`--sequences-file` contains real Khitan Small Script character sequences, one sequence per line.

Only sequences with one to seven signs are used. A sequence is skipped if any sign is missing from the crop bank.

### Output

The output structure is the same as the random dynamic layout dataset:

```text
data/synth_real_layout/
├── train/
│   ├── images/
│   └── labels.json
├── val/
│   ├── images/
│   └── labels.json
└── test/
    ├── images/
    └── labels.json
```

### Main arguments

| Argument                  | Description                               |
| ------------------------- | ----------------------------------------- |
| `--bank-dir`              | Input sign crop bank                      |
| `--output-root`           | Output dataset directory                  |
| `--radicals-file`         | Character inventory file                  |
| `--sequences-file`        | Real sequence file                        |
| `--augmentations-per-seq` | Number of augmented samples per sequence  |
| `--img-size`              | Output image size, default is `256`       |
| `--block-min`             | Minimum character block size              |
| `--block-max`             | Maximum character block size              |
| `--train-ratio`           | Training split ratio, default is `0.90`   |
| `--val-ratio`             | Validation split ratio, default is `0.05` |
| `--seed`                  | Random seed for reproducibility           |
| `--clear-output`          | Remove existing output before generation  |

## 4. Degradation Ablation Synthesis

This script generates sequence-based synthetic datasets with controlled degradation levels. It is mainly used for ablation experiments on robustness to rubbing-style degradation.

```bash
python create_real_dataset_by_degradation.py \
  --bank-dir dataset/radical_bank \
  --output-base dataset/degradation_ablation \
  --radicals-file assets/radicals.txt \
  --sequences-file dataset/sequences.csv \
  --level L2_Moderate \
  --clear-output
```

### Supported degradation levels

| Level         | Description          |
| ------------- | -------------------- |
| `L1_Mild`     | Mild degradation     |
| `L2_Moderate` | Moderate degradation |
| `L3_Severe`   | Severe degradation   |

The degradation process includes local stroke corruption, elastic deformation, perspective distortion, noise, blur, white scratches, black ink stains, and global image degradation.

### Output

The selected degradation level is used as a subdirectory under `--output-base`:

```text
data/degradation_ablation/
└── L2_Moderate/
    ├── train/
    │   ├── images/
    │   └── labels.json
    ├── val/
    │   ├── images/
    │   └── labels.json
    └── test/
        ├── images/
        └── labels.json
```

### Main arguments

| Argument                  | Description                                                 |
| ------------------------- | ----------------------------------------------------------- |
| `--bank-dir`              | Input sign crop bank                                        |
| `--output-base`           | Base output directory                                       |
| `--radicals-file`         | Character inventory file                                    |
| `--sequences-file`        | Real sequence file                                          |
| `--level`                 | Degradation level: `L1_Mild`, `L2_Moderate`, or `L3_Severe` |
| `--augmentations-per-seq` | Number of augmented samples per sequence                    |
| `--img-size`              | Output image size, default is `256`                         |
| `--block-min`             | Minimum character block size                                |
| `--block-max`             | Maximum character block size                                |
| `--train-ratio`           | Training split ratio, default is `0.90`                     |
| `--val-ratio`             | Validation split ratio, default is `0.05`                   |
| `--clear-output`          | Remove existing output before generation                    |

## Notes

- All paths in the examples are relative paths and can be modified according to the project structure.
- The scripts do not contain machine-specific absolute paths.
- The generated `labels.json` files use normalized center-size bounding boxes.
- Use `--clear-output` carefully, because it removes the existing output directory before generating new data.
- For quick testing, reduce `--num-images` or `--augmentations-per-seq`.
